Event loop in Java Script

JavaScript is single-threaded, meaning it executes one thing at a time. But still, it can handle asynchronous tasks (like setTimeout, API calls, promises, etc.) without blocking

How it works.

1.Call Stacks - Js executes code line by line here.

2.Web APIs (Browser APIs) - When we use function like setTimeout, fetch, etc., the browser takes care of them separately.

3.Callback Queue (Task Queue / Message Queue) - When the browser finishes those async tasks, their callbacks are placed in the queue.

4.Event Loop - Constantly checks:
i."Is the call stack is empty?"
ii. If yes, it pushes the first callback from the queue into the call stack.

CallBack Queue.

    It stores callbacks waiting to be executed after async tasks complete.

    Example: setTimeout, setInterval, DOM events, etc.

    The event loop takes these one by one and puts them on the call stack to run.
