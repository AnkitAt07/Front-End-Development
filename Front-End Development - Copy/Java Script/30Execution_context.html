Execution Context in JavaScript

Understanding how JavaScript code is executed under the hood

What is Execution Context?

An Execution Context is an abstract concept that refers to the environment in which JavaScript code is evaluated and executed. Think of it as a "workspace" or a "container" that holds all the information necessary to run a specific piece of code, such as variables, functions, and the scope chain.

Whenever any code is run in JavaScript, it's run inside an execution context.

Types of Execution Contexts

There are three main types of execution contexts:

1. Global Execution Context (GEC)

  • This is the default or base context
  • It is created once when the JavaScript script first starts to execute
  • It's associated with the global object (window in browsers, global in Node.js)
  • All global variables and functions are stored inside the GEC

2. Function Execution Context (FEC)

  • A new FEC is created every time a function is invoked (called)
  • Each function call gets its own independent execution context
  • The code inside the function is executed within this context

3. Eval Execution Context

  • Code executed inside the eval() function gets its own context
  • Generally considered dangerous and bad practice
  • Rarely used in modern JavaScript

The Execution Context Stack (Call Stack)

Since JavaScript is single-threaded, it can only execute one thing at a time. To manage the creation and execution of these multiple contexts, the JavaScript engine uses a Call Stack (also known as the Execution Context Stack).

function firstFunction() {
  console.log("This is first function");
  secondFunction(); // Call second function
}

function secondFunction() {
  console.log("This is second function");
}

firstFunction(); // Call first function
console.log("This is global context");

How the Call Stack Works

  • The stack is LIFO (Last-In, First-Out)
  • When the script loads, the Global EC is created and pushed onto the stack
  • When a function is called, a new Function EC is pushed on top of the stack
  • When the function finishes, its EC is popped off the stack
  • This continues until the global context is popped off

Phases of Execution Context

Phase 1: Creation Phase (Hoisting)

Before code execution, the JavaScript engine sets up the context:

  • Creates the Variable Object (VO) (also called the Lexical Environment)
  • Scans for function declarations - creates properties in VO pointing to functions
  • Scans for variable declarations - creates properties in VO initialized to undefined
  • Determines the value of this
  • Creates the Scope Chain

Phase 2: Execution Phase

The code is executed line-by-line:

  • Values are assigned to variables
  • Functions are executed (creating new Function ECs)

Hoisting Example

console.log(myVar); // Output: undefined
console.log(sayHello()); // Output: "Hello!"

var myVar = 10;

function sayHello() {
  return "Hello!";
}

Summary

Concept Description
Execution Context The environment where JS code is executed. Contains all necessary information (variables, functions, scope).
Global EC The base context for all code not inside a function. Created once.
Function EC A new context created every time a function is called.
Call Stack A LIFO stack that manages the order of execution contexts.
Creation Phase Sets up the Variable Object (hoisting), determines this, and creates the Scope Chain.
Execution Phase Runs the code line-by-line, assigning values and executing functions.
